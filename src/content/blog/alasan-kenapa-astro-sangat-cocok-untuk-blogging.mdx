Beberapa waktu lalu, saya akhirnya memutuskan untuk **mengubah total** platform blog personal ini. Stack lama yang berbasis **Laravel** saya tinggalkan, lalu saya rebuild ulang menggunakan **Astro**.

Awalnya, alasannya sederhana—bahkan agak pragmatis.
Saya ingin **menghemat biaya VPS**, dan jujur saja, saya mulai merasa stack lama terlalu “berat” untuk sesuatu yang pada akhirnya hanya saya pakai untuk menulis.

Blog ini tidak butuh queue, tidak butuh auth kompleks, dan jelas tidak butuh database yang hidup 24 jam. Yang saya butuhkan cuma satu: **tempat menulis yang nyaman dan cepat diakses**.

Dan dari situ, Astro mulai terasa sangat masuk akal.


## Blog Itu Harusnya Tentang Menulis, Bukan Maintenance

Setelah beberapa tahun ngoprek berbagai stack, saya sampai pada satu kesimpulan kecil:
**blog personal itu seharusnya tidak melelahkan untuk dirawat**.

Dengan Astro, saya benar-benar bisa fokus ke konten.
Tidak ada server yang harus dipantau. Tidak ada error random dari background process. Tidak ada rasa “nanti aja nulisnya, lagi males ngurusin teknis”.

Saya buka editor, nulis, commit, deploy. Selesai.

Dan untuk saya pribadi, itu priceless.

## Nulis Pakai Markdown

Astro mendukung Markdown dan MDX secara native, dan ini salah satu poin yang paling saya nikmati.

Sebagai developer, Markdown Itu udah jadi konsumsi sehari - hari para developer.
Tidak perlu mikir styling. Tidak perlu mikir layout berlebihan. Fokus ke isi.

Yang bikin makin enak, MDX memungkinkan saya:

* Nyelipin komponen kecil di tengah artikel
* Bikin highlight atau callout tanpa ribet
* Tetap menjaga artikel terasa hidup, bukan sekadar teks polos

Menulis jadi terasa natural, bukan seperti “ngoding halaman”.

## Performa Cepat Tanpa Saya Harus Sok Jago Optimasi

Salah satu hal yang bikin saya terkejut adalah:
**blog ini sudah cepat bahkan sebelum saya niat optimasi**.

Astro secara default tidak mengirim JavaScript kalau memang tidak dibutuhkan.
Hasilnya?

* Halaman kebuka instan
* Scroll terasa ringan
* Tidak ada drama loading aneh-aneh

Sebagai pembaca, ini nyaman.
Sebagai developer, ini bikin tenang—karena saya tidak harus terus-terusan mikirin performa.


## SEO yang Rasanya “Masuk Akal”

Biasanya, ngurus SEO itu terasa seperti kerjaan tambahan.
Dengan Astro, rasanya lebih seperti bagian alami dari proses.

Karena file `.astro` itu pada dasarnya JSX + HTML, saya punya kontrol penuh:

* Meta tag
* Open Graph
* Struktur heading
* Semantic HTML

Tidak ada magic berlebihan. Tidak ada abstraksi aneh.
Apa yang saya tulis, itu yang dirender.


## Hosting Gratis

Ini bagian favorit saya.

Setelah pindah ke Astro:

* Tidak perlu VPS
* Tidak perlu server always-on
* Tinggal deploy ke static hosting seperti netlify, github pages atau cloudflare pages

Secara biaya, ini game changer.
Secara mental, ini bikin lega—tidak ada rasa “harus kepake biar balik modal”.

Blog ini akhirnya benar-benar jadi **ruang pribadi**, bukan aset yang harus dioptimalkan terus.



## Developer Experience yang Bikin Betah

Astro tidak maksa saya pakai satu framework.
Kalau mau React, bisa.
Kalau mau vanilla, bisa.
Kalau mau campur, juga bisa.

Struktur project-nya jelas, build-nya cepat, dan tidak banyak hal “ajaib” di balik layar.

Buat saya, ini penting.
Blog ini bukan cuma tempat nulis, tapi juga tempat eksperimen kecil tanpa tekanan.



## Penutup

Setelah migrasi ke Astro, satu hal yang paling terasa adalah **saya jadi lebih sering nulis**.

Bukan karena Astro canggih, tapi karena:

* Beban teknis berkurang
* Proses nulis jadi sederhana
* Tidak ada alasan untuk menunda

Dan mungkin itu tujuan utama dari sebuah blog personal.

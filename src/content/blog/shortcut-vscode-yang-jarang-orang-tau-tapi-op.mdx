## Kenyataan Developer Modern yang Sering Tidak Disadari

Banyak developer muda merasa sudah produktif karena bisa menyelesaikan fitur.  
 Kodenya jalan. Build sukses. Deploy aman.

Tapi kalau diperhatikan lebih dalam, ada satu kebiasaan yang diam diam menghambat progres mereka.

Terlalu sering pegang mouse.

Scroll pakai mouse.  
 Klik menu satu satu.  
 Rename variabel manual.  
 Edit baris berulang dengan cara yang sama.

Masalahnya bukan di skill ngoding, tapi **cara berinteraksi dengan editor**.

VSCode itu bukan sekadar text editor. Ia adalah alat kerja.  
 Dan seperti alat kerja lainnya, **siapa yang tahu cara pakainya dengan benar akan bekerja jauh lebih cepat**.

Artikel ini akan membahas shortcut VSCode yang:

*   Jarang disadari pemula
*   Terlihat sederhana
*   Tapi efeknya brutal ke produktivitas

Fokus kita:

*   Multi cursor
*   Command palette
*   Rename symbol

Kalau kamu menguasai tiga ini, workflow kamu akan naik kelas tanpa harus ganti laptop atau begadang.

## Kenapa Shortcut VSCode Itu Penting Banget

Coba jujur sebentar.

Dalam satu jam ngoding, berapa menit yang benar benar dipakai untuk mikir logic  
 dan berapa menit yang habis buat:

*   Scroll
*   Klik
*   Cari menu
*   Rename satu satu

Banyak developer tidak sadar bahwa **waktu habis bukan karena kode susah, tapi navigasi lambat**.

Shortcut itu penting karena:

*   Mengurangi distraksi
*   Menjaga fokus tetap di kode
*   Menghemat energi mental

Developer yang terbiasa pakai shortcut biasanya:

*   Lebih tenang
*   Lebih rapi
*   Lebih konsisten

Bukan karena mereka jenius, tapi karena **workflow mereka efisien**.

Dan di dunia kerja, efisiensi itu mahal nilainya.

## Multi Cursor

### Apa Itu Multi Cursor

Multi cursor adalah fitur yang memungkinkan kamu **mengedit banyak posisi sekaligus dalam satu waktu**.

Bayangkan kamu bisa:

*   Ngetik satu kali
*   Tapi perubahan terjadi di banyak baris

Buat pemula, ini sering terasa seperti cheat.  
 Buat developer berpengalaman, ini kebutuhan sehari hari.

Multi cursor bukan trik pamer.  
 Ini alat serius untuk kerja cepat.

### Cara Pakai Multi Cursor yang Paling Sering Dipakai

Ada beberapa cara umum yang sering dipakai.

Pertama, memilih kata yang sama di banyak tempat.  
 Cocok untuk:

*   Variabel
*   Class
*   Function name

Kedua, menambah cursor manual di beberapa baris.  
 Cocok untuk:

*   Edit struktur HTML
*   Tambah properti
*   Edit data berurutan

Ketiga, edit vertikal.  
 Ini sering dipakai saat:

*   Menyamakan indent
*   Mengubah format data
*   Menyusun array atau object

Begitu terbiasa, tangan kamu akan otomatis ke keyboard tanpa mikir lagi.

### Use Case Nyata Multi Cursor di Dunia Kerja

Multi cursor sering dipakai untuk:

*   Rename variabel di satu file dengan cepat
*   Edit banyak atribut HTML sekaligus
*   Mengubah struktur JSON atau config

Contoh nyata:  
 Kamu punya 20 baris HTML dengan class yang mirip.  
 Tanpa multi cursor, kamu edit satu satu.  
 Dengan multi cursor, selesai dalam hitungan detik.

Di sinilah beda developer yang:

*   Kerja keras  
     dan
*   Kerja cerdas

### Kesalahan Umum Saat Pakai Multi Cursor

Walaupun powerful, multi cursor juga bisa berbahaya kalau asal pakai.

Kesalahan umum:

*   Edit terlalu banyak tanpa cek konteks
*   Mengubah kata yang seharusnya tidak diubah
*   Lupa scope kode

Multi cursor bukan pengganti logika.  
 Dia cuma mempercepat tangan, bukan menggantikan otak.

Biasakan:

*   Cek hasil akhir
*   Scroll ulang setelah edit
*   Jangan terlalu agresif

## Command Palette

### Apa Itu Command Palette

Command palette adalah pintu rahasia VSCode yang jarang dimanfaatkan maksimal.

Semua fitur VSCode  
 dari yang kelihatan sampai yang tersembunyi  
 bisa diakses dari satu tempat.

Daripada:

*   Cari menu
*   Buka setting
*   Scroll panjang

Kamu cukup:

*   Buka command palette
*   Ketik keyword
*   Jalankan perintah

Ini adalah shortcut paling underrated di VSCode.

### Kenapa Command Palette Terasa OP

Command palette itu seperti search engine untuk VSCode.

Kamu tidak perlu hafal:

*   Di mana menu berada
*   Di tab apa setting disimpan

Cukup tahu **kata kuncinya**.

Ini membuat kamu:

*   Lebih cepat
*   Lebih fleksibel
*   Lebih fokus ke tujuan

Developer berpengalaman jarang buka menu manual.  
 Mereka cari semuanya lewat command palette.

### Command Palette untuk Kerja Cepat

Beberapa hal yang sering dilakukan lewat command palette:

*   Buka file dengan cepat
*   Ganti tema editor
*   Masuk ke setting
*   Restart language server

Semua tanpa mouse.

Kalau kamu kerja di project besar, ini sangat terasa bedanya.

### Command Palette untuk Workflow Pro

Di level lebih lanjut, command palette dipakai untuk:

*   Menjalankan fitur extension
*   Mengakses command tersembunyi
*   Navigasi fitur advanced VSCode

Banyak fitur VSCode tidak punya shortcut default.  
 Satu satunya cara cepat mengaksesnya adalah lewat command palette.

Ini alasan kenapa developer senior terlihat seperti:  
 “Ngetik bentar, kok langsung kebuka”

### Kebiasaan Kecil yang Bikin Beda

Developer pemula sering bertanya:  
 “Shortcut ini apa ya”

Developer berpengalaman berpikir:  
 “Apa keyword command nya”

Mereka tidak hafal semua shortcut.  
 Mereka hafal **cara mencari fitur dengan cepat**.

Command palette membentuk mindset ini.

## Rename Symbol

### Apa Itu Rename Symbol

Rename symbol adalah fitur untuk **mengganti nama variabel, function, atau class dengan aman dan pintar**.

Ini berbeda dengan rename manual atau search replace.

Rename symbol:

*   Paham scope
*   Paham struktur kode
*   Tidak asal ganti teks

Fitur ini bekerja berkat language server yang memahami bahasa pemrograman kamu.

### Kenapa Rename Symbol Itu OP

Rename symbol itu:

*   Aman
*   Konsisten
*   Minim risiko error

Kamu bisa:

*   Rename function besar
*   Rename class global
*   Rename variabel penting

Tanpa takut:

*   Typo
*   Salah scope
*   Merusak bagian lain

Ini sangat penting di project besar.

### Use Case Rename Symbol di Project Nyata

Rename symbol sering dipakai saat:

*   Refactor kode lama
*   Mengganti nama variabel yang ambigu
*   Menyesuaikan naming convention tim

Bayangkan rename manual di puluhan file.  
 Itu mimpi buruk.

Rename symbol mengubah mimpi buruk jadi pekerjaan satu menit.

### Kesalahan Umum Developer Pemula

Kesalahan yang sering terjadi:

*   Rename manual satu satu
*   Pakai search replace global
*   Takut refactor karena takut error

Padahal rename symbol dibuat justru untuk menghindari hal hal itu.

Kalau kamu masih rename variabel penting secara manual,  
 itu tanda kamu belum memanfaatkan editor secara maksimal.

## Kombinasi Shortcut yang Bikin Kamu Terlihat Pro

Yang bikin workflow kamu naik level bukan satu shortcut, tapi **kombinasi kebiasaan**.

Contoh workflow nyata:

*   Gunakan command palette untuk navigasi
*   Gunakan rename symbol untuk refactor
*   Gunakan multi cursor untuk edit cepat

Tanpa mouse.  
 Tanpa panik.  
 Tanpa ribet.

Di mata orang lain, ini terlihat seperti:  
 “Cepat banget kerjanya”

Padahal kamu cuma pakai alat dengan benar.
